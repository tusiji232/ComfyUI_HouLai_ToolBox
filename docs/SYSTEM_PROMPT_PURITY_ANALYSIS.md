# 系统提示词"污染"问题分析

## 问题识别

用户提出的concern：系统提示词中的以下表述是否构成"污染"：

```markdown
- 产品图显示是**中式盘扣** → 必须使用 `"Chinese frog button closure"`
- 技能文件默认是 `"horn button"` → **完全忽略，使用提取值**
```

**"污染"的定义**：
系统提示词包含了本应由**产品图**或**技能文件**提供的具体细节，导致：
1. 系统提示词失去通用性
2. 可能限制LLM对其他产品的正确处理
3. 与技能文件的职责边界模糊

---

## 污染程度评估

### 🔴 确实构成轻度污染

| 污染类型 | 具体表现 | 影响 |
|----------|----------|------|
| **具体案例污染** | "中式盘扣"、"horn button" | 让系统提示词带上特定产品痕迹 |
| **隐式偏好污染** | 使用盘扣作为"冲突解决"的示例 | 可能让LLM过度关注盘扣类问题 |

### 🟡 但污染程度有限

这些例子属于**教学性示例**，不是硬性规则：
- 没有说"所有产品都必须检查是否有盘扣"
- 只是用盘扣来说明"提取值优先于默认值"的原则
- LLM应该能理解这是示例而非限制

---

## 优化方案

### 方案A：将具体例子改为通用表述（推荐）

**当前写法（有污染）**：
```markdown
**规则1：提取值绝对优先**
- 产品图显示是**中式盘扣** → 必须使用 `"Chinese frog button closure"`
- 技能文件默认是 `"horn button"` → **完全忽略，使用提取值**
- 产品图显示是**短款** → 使用 `"hip-length"` 或 `"short"`
- 技能文件描述是 `"long coat"` → **完全忽略，使用提取值**
```

**优化写法（无污染）**：
```markdown
**规则1：提取值绝对优先**

当产品图提取的信息与技能文件默认值冲突时：
- **始终使用产品图提取的值**
- **完全忽略技能文件的默认值**

示例（这些只是示例，不是固定规则）：
- 产品图显示是A类型扣子 → 使用 "A类型扣子"，忽略技能文件的"B类型扣子"
- 产品图显示是短款 → 使用 "short/hip-length"，忽略技能文件的"long"
```

### 方案B：明确标注"示例"属性

如果不想删除具体例子，可以明确标注：

```markdown
**规则1：提取值绝对优先**

无论产品图显示什么具体细节，都优先使用产品图的描述：
- ✅ 使用产品图提取的具体特征（如：中式盘扣、毛绒领、短款等）
- ❌ 忽略技能文件中冲突的默认值（如：horn button、西装领、长款等）

*注：以上括号中的内容仅为示例，实际应根据产品图提取的具体信息填写*
```

### 方案C：完全删除具体例子，仅保留原则

**最纯净的写法**：
```markdown
**规则1：提取值绝对优先**

产品图分析提取的所有细节（款式、面料、颜色、装饰元素等）具有最高优先级。

执行规则：
1. 完整提取产品图中的所有可见特征
2. 与技能文件模板对比，识别冲突项
3. 在所有冲突中，使用产品图提取的信息
4. 完全忽略技能文件中与产品图冲突的默认值

自检问题：
- 技能文件说"长款"，但产品图是短款？→ 使用"短款"
- 技能文件说"纽扣"，但产品图是"盘扣/拉链/钩子"？→ 使用产品图的实际门襟类型
- 技能文件说"红色"，但产品图是"蓝色"？→ 使用"蓝色"
```

---

## 推荐方案：方案C（完全去示例化）

理由：
1. **最纯净**：系统提示词完全不包含任何产品细节
2. **最通用**：适用于任何类型的产品
3. **职责清晰**：产品细节100%来自产品图，系统提示词只提供规则框架

---

## 系统提示词修正版本（去污染版）

### 修正后的【原则1：产品细节保真】

```markdown
### 原则1：产品细节保真（第一优先级）

**产品图提取的信息是绝对权威，技能文件只能补充，不能覆盖。**

#### 必须准确提取的产品细节

分析产品图时必须完整识别：

| 细节类别 | 提取内容 |
|----------|----------|
| **款式特征** | 版型、长度、剪裁 |
| **面料质感** | 光泽、纹理、厚度 |
| **颜色色调** | 固有色（排除光线影响）|
| **门襟类型** | 扣子/拉链/盘扣/其他 |
| **领型设计** | 立领/翻领/围巾领/其他 |
| **袖口处理** | 直筒/收紧/翻边/毛绒/其他 |
| **口袋设计** | 贴袋/插袋/斜插袋/无 |
| **装饰元素** | 刺绣/拼接/滚边/其他 |

#### 产品细节保护规则

**规则：提取值绝对优先**

产品图分析提取的所有细节具有最高优先级，技能文件默认值必须让路。

执行流程：
1. 完整提取产品图中的所有可见特征
2. 与技能文件模板对比，识别冲突项
3. 在所有冲突中，**强制使用产品图提取的信息**
4. **完全忽略**技能文件中与产品图冲突的默认值

冲突解决示例（这些仅为说明原则，非固定内容）：
- 技能文件说"长款"，产品图是短款 → 使用"短款"
- 技能文件说"纽扣门襟"，产品图是"盘扣/拉链" → 使用产品图的实际门襟
- 技能文件说"红色"，产品图是"蓝色" → 使用"蓝色"

**禁止行为**：
- ❌ 绝不允许：产品图有特殊设计但模板没定义，就简化为通用描述
- ❌ 绝不允许：产品图有明确特征，就用技能文件默认值覆盖
- ✅ 必须：准确描述产品图显示的所有特征，无论技能文件是否定义
```

---

## 总结

### 污染判定

| 内容 | 污染程度 | 说明 |
|------|----------|------|
| "中式盘扣"、"horn button" 作为示例 | 🟡 轻度污染 | 是示例但过于具体 |
| "提取值绝对优先"原则本身 | 🟢 无污染 | 这是系统提示词应有的规则 |

### 建议行动

**立即修改**：将具体产品示例（中式盘扣、horn button）改为通用表述或删除，保持系统提示词的纯净性。

**修改后的职责边界**：

```
系统提示词（纯净框架）
├── 原则：产品细节保真
├── 规则：提取值优先于默认值
├── 方法：冲突解决机制
└── ❌ 不包含任何具体产品细节

产品图（细节来源）
├── 具体款式特征
├── 具体面料颜色
├── 具体设计元素（盘扣/毛绒领等）
└── ✅ 所有细节从这里提取

技能文件（构图参考）
├── 构图框架
├── 光线背景
├── 质量参数
└── ⚠️ 默认值可能被产品图覆盖
```

这样系统提示词就真正做到了"指导框架"的定位，没有任何产品细节的"污染"。
