# ComfyUI_HouLai_ToolBox - LLM 技能系统技术文档

## 📖 概述

LLM 技能系统是 `ComfyUI_HouLai_ToolBox` 插件的核心功能模块，提供基于 YAML 配置文件的智能提示词生成能力。该系统允许用户通过预定义的技能模板，快速生成适合电商产品展示的 AI 绘图提示词。

## 🔧 核心组件详解

### 1. 后端节点 (houlai_llm_agent.py)

#### 1.1 Universal_LLM_Config 节点

**功能**: 配置 LLM 连接参数

**输入参数**:
| 参数名 | 类型 | 说明 |
|--------|------|------|
| `base_url` | STRING | API 基础 URL，如豆包或 OpenAI 地址 |
| `api_key` | STRING | API 密钥（密码隐藏显示） |
| `model_name` | STRING | 模型名称，如 gpt-4o、ep-xxx |
| `system_prompt` | STRING | 系统级提示词，定义 AI 角色 |

**输出**:
- `llm_config` (LLM_CONFIG): 配置对象，传递给下游节点

---

#### 1.2 Ecommerce_Skill_Router 节点

**功能**: 根据技能模板生成批量提示词

**必需参数**:
| 参数名 | 类型 | 说明 |
|--------|------|------|
| `技能选择` | COMBO | 选择预设的技能模板 |
| `LLM配置` | LLM_CONFIG | 上游 LLM 配置节点输出 |
| `输出模式` | COMBO | 分批输出/合并输出 |
| `生图数量` | INT | 需要生成的图片数量 (1-20) |
| `使用技能` | BOOLEAN | 开关：使用 YAML 技能或自定义模板 |

**可选参数**:
| 参数名 | 类型 | 说明 |
|--------|------|------|
| `关键词搜索` | STRING | 输入关键词自动匹配技能 |
| `自定义技能目录` | STRING | 指定自定义 skills 目录路径 |
| `图片1-4` | IMAGE | 输入产品参考图片（最多4张） |
| `产品名称` | STRING | 产品名称 |
| `目标人群` | STRING | 目标用户群体 |
| `产品参数` | STRING | 产品规格参数 |
| `卖点` | STRING | 产品卖点描述 |
| `平台` | STRING | 目标电商平台 |
| `语言` | STRING | 输出语言 |
| `自定义模板` | STRING | 关闭技能时使用此模板 |

**输出**:
- `batch_prompts` (STRING[]): 生成的提示词列表（列表类型，可触发批量任务）
- `formatted_summary` (STRING): 完整输出文本汇总

---

### 2. 前端扩展 (houlai_dynamic_skills.js)

#### 2.1 主要功能

1. **动态技能列表加载**
   - 节点创建时自动从后端获取技能列表
   - 支持自定义技能目录

2. **触发词自动匹配**
   - 监听"关键词搜索"输入
   - 根据触发词自动选择匹配的技能
   - 500ms 防抖处理

3. **技能目录变更监听**
   - 自定义技能目录变化时自动刷新列表
   - 清空缓存确保数据最新

#### 2.2 核心函数

```javascript
// 从后端获取技能列表
async function fetchSkills(customPath = "")

// 更新节点的技能选择下拉菜单
function updateSkillDropdown(node, skills)

// 根据触发词搜索匹配的技能
function matchSkillByTrigger(text, triggerMap)

// 为节点设置动态技能功能
function setupDynamicSkills(node)
```

### 3. 技能文件格式 (YAML)

#### 3.1 文件结构

```yaml
# ============================================
# {商品大类}产品技能配置文件
# 适用于: {具体商品列举}
# 变量说明:
#   {platform} - 目标电商平台
#   {selling_points} - 产品卖点描述
#   {batch_count} - 需要生成的提示词数量
# ============================================

# ------------------- {技能名称1} -------------------
Skill_Key_Name:
  name: "{中文显示名称}"
  description: "{技能功能描述}"
  category: "{商品分类}"
  triggers: ["{触发词1}", "{触发词2}"]
  template: |
    你是一位{专业角色定位}，精通 {platform} 平台的{专业领域}。
    
    请为一款{商品名称}生成 {batch_count} 个{内容类型}的AI绘图提示词。
    
    【产品卖点】
    {selling_points}
    
    【{要求类型}要求】
    1. 第一张：{具体描述}
    2. 第二张：{具体描述}
    3. 第三张：{具体描述}
    4. 第四张：{具体描述}
    
    【输出格式】
    - 每行一条独立的英文提示词
    - 强调{核心视觉要素}
    - 适合 Flux/Qwen 等绘图模型使用
```

#### 3.2 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | STRING | 技能的显示名称（中文） |
| `description` | STRING | 技能描述说明 |
| `category` | STRING | 商品分类标签 |
| `triggers` | STRING[] | 触发关键词列表 |
| `template` | STRING | 提示词模板（支持变量插值） |

#### 3.3 模板变量

| 变量名 | 说明 | 来源 |
|--------|------|------|
| `{platform}` | 目标平台名称 | 节点"平台"输入 |
| `{selling_points}` | 产品信息上下文 | 节点各产品字段拼接 |
| `{batch_count}` | 生成数量 | 节点"生图数量"参数 |

## 🔄 工作流程

### 1. 节点初始化流程

```
用户添加 Ecommerce_Skill_Router 节点
         │
         ▼
前端扩展 houlai_dynamic_skills.js 捕获节点创建事件
         │
         ▼
调用 fetchSkills() 从后端获取技能列表
         │
         ▼
更新节点"技能选择"下拉菜单选项
         │
         ▼
监听"关键词搜索"和"自定义技能目录"变化
```

### 2. 执行流程

```
用户点击执行
       │
       ▼
Ecommerce_Skill_Router.process() 被调用
       │
       ├─ 1. 处理关键词搜索（触发词匹配）
       │
       ├─ 2. 构建产品信息上下文
       │      (产品名称、卖点、平台等)
       │
       ├─ 3. 加载技能模板（根据"使用技能"开关）
       │      - 开启: 从 YAML 文件加载
       │      - 关闭: 使用"自定义模板"
       │
       ├─ 4. 格式化模板（替换变量）
       │
       ├─ 5. 处理输入图片（tensor → base64）
       │
       ├─ 6. 调用 LLM API
       │
       └─ 7. 解析响应，返回提示词列表
```

### 3. 触发词匹配流程

```
用户在"关键词搜索"输入文本
            │
            ▼
500ms 防抖处理
            │
            ▼
调用 matchSkillByTrigger(text, triggerMap)
            │
            ├─ 文本转小写
            ├─ 遍历触发词映射
            └─ 查找包含关系
            │
            ▼
      找到匹配？
       /        \
     是          否
     │            │
     ▼            ▼
自动选择技能   保持原选择
显示 Toast 提示
```

## 🛠️ API 接口

### 1. 获取技能列表

**Endpoint**: `GET /houlai/get_skills`

**Query Parameters**:
| 参数名 | 类型 | 说明 |
|--------|------|------|
| `custom_path` | STRING | 自定义技能目录路径（可选） |

**Response**:
```json
{
  "success": true,
  "skills": ["beauty - Lipstick_Detail", "digital - Phone_Product"],
  "triggers": {
    "口红": "beauty - Lipstick_Detail",
    "手机": "digital - Phone_Product"
  }
}
```

### 2. 刷新技能列表

**Endpoint**: `POST /houlai/refresh_skills`

**Request Body**:
```json
{
  "custom_path": "/path/to/custom/skills"
}
```

## 📝 技能开发指南

### 1. 创建新技能文件

在 `skills/` 目录下创建 YAML 文件：

```bash
touch skills/my_category.yaml
```

### 2. 编写技能配置

参考以下模板：

```yaml
# 食品类产品技能配置
Food_Product:
  name: "食品产品图"
  description: "生成食品产品展示图"
  category: "食品"
  triggers: ["食品", "food", "零食", "美食"]
  template: |
    你是一位美食摄影师，精通 {platform} 平台的食物摄影。
    
    请为一款食品生成 {batch_count} 个产品展示图提示词。
    
    【产品卖点】
    {selling_points}
    
    【展示要求】
    1. 第一张：产品正面，突出色泽和质感
    2. 第二张：细节特写，展示纹理和新鲜度
    3. 第三张：场景搭配，餐具和环境
    4. 第四张：创意摆盘，艺术感构图
    
    【输出格式】
    - 每行一条独立的英文提示词
    - 强调食欲感和新鲜感
    - 适合 Flux/Qwen 等绘图模型使用
```

### 3. 技能设计原则

1. **专业角色定位**: 根据商品类型选择合适的专业角色
   - 时尚类 → 时尚摄影师、造型师
   - 科技类 → 产品摄影师、3C专家
   - 食品类 → 美食摄影师、餐饮视觉设计师
   - 家居类 → 室内设计师、生活方式摄影师

2. **展示角度设计**: 每个技能包含4个不同角度
   - 第一张：产品主体展示（正面/特写）
   - 第二张：细节/质感展示（材质/工艺）
   - 第三张：场景应用（使用环境）
   - 第四张：创意/氛围图（情感传达）

3. **触发词设置**: 
   - 包含中英文关键词
   - 覆盖产品别称
   - 考虑用户习惯用语

## 🔍 故障排查

### 1. 技能列表不显示

**可能原因**:
- `skills/` 目录不存在或为空
- YAML 文件语法错误
- API 端点未正确注册

**排查步骤**:
1. 检查 `skills/` 目录是否存在
2. 验证 YAML 文件语法（使用 yaml linter）
3. 查看 ComfyUI 控制台日志

### 2. 触发词匹配失败

**可能原因**:
- 触发词大小写问题（实际已转小写处理）
- 技能文件未正确加载
- 缓存未刷新

**解决方法**:
1. 修改"自定义技能目录"触发重新加载
2. 重启 ComfyUI

### 3. LLM 调用失败

**可能原因**:
- API Key 无效
- 网络连接问题
- 模型名称错误

**排查步骤**:
1. 验证 API Key 是否正确
2. 检查 base_url 是否可访问
3. 确认模型名称在服务商处有效
