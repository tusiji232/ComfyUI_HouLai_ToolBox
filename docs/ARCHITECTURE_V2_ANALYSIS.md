# 系统提示词与技能文件架构关系分析报告 V2

> 核心原则：系统提示词 = 指导框架（无预设），技能文件 = 执行标准（唯一权威）
> 输出目标：单行提示词，支持批量出图
> 关键能力：产品图 + 技能文件 = 还原视觉效果

---

## 一、架构定位重新定义

### 1.1 三层角色清晰划分

```
┌─────────────────────────────────────────────────────────────────┐
│  【输入层】产品图 + 基础参数                                      │
│  - 用户上传的产品参考图                                           │
│  - 平台选择（淘宝/京东/亚马逊等）                                 │
│  - 语言选择（zh-CN/en/ja等）                                     │
│  - 生图数量（batch_count）                                        │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  【指导层】系统提示词（轻量级框架）                                │
│  职责：告诉LLM如何"使用技能文件"                                 │
│  - 如何分析产品图提取关键信息                                     │
│  - 如何匹配技能文件中的变量                                       │
│  - 如何保持批量输出格式一致性                                     │
│  ❌ 不包含：任何固定提示词、任何风格预设                          │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  【执行层】技能文件（唯一权威）                                    │
│  职责：提供可执行的提示词模板                                     │
│  - shots: 8个固定机位的英文提示词                                │
│  - template: 变量替换规则                                        │
│  - defaults: 默认值配置                                          │
│  ✅ 所有视觉风格、构图、光线定义都在这里                           │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  【输出层】批量提示词列表                                         │
│  - 每行1条提示词                                                 │
│  - 可直接用于 Flux/Midjourney/SD 批量出图                        │
│  - 保持技能文件定义的视觉效果                                    │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 核心设计哲学

| 层级 | 设计原则 | 关键特征 |
|------|----------|----------|
| 系统提示词 | **指导性、无状态** | 只教"怎么做"，不说"做什么" |
| 技能文件 | **权威性、可执行** | 只提供"可替换的模板" |
| 产品图 | **输入源、可变** | 提供"填充模板的素材" |

## 二、系统提示词应该教什么？

### 2.1 当前系统提示词的问题

**问题1：包含过多预设内容**

当前系统提示词（第54-77行）：
```markdown
## 【输出要求】

### 提示词结构
每条提示词必须包含以下要素：
1. **主体描述**：产品/模特/场景的核心描述
2. **文字排版区域**：明确标注标题区、卖点区、标签区位置
3. **构图布局**：layout, composition, aspect ratio
4. **光影效果**：lighting style, shadow, highlight
5. **风格修饰**：platform style, aesthetic keywords
6. **质量参数**：4K, high quality, professional photography
```

**问题**：这些结构要求可能与技能文件的实际shots冲突。例如技能文件已经定义了具体的光线和构图，系统不应该再要求LLM重新决定。

**问题2：越俎代庖定义视觉元素**

系统提示词中定义了：
- 字体规范
- 色彩体系
- 摄影参数

这些应该完全由技能文件决定。

### 2.2 优化后的系统提示词职责

```markdown
## 【你的任务】

你是一个"技能文件执行器"。你的唯一工作是将用户的产品图信息，
填充到技能文件提供的模板中，生成批量提示词。

## 【执行步骤】

### 步骤1：分析产品图
从产品图中提取以下信息，用于填充技能文件变量：
- 产品类型（clothing_item）: 根据图中服装判断
- 面料类型（fabric_type）: 观察材质光泽、纹理
- 主要颜色（color）: 提取主色调
- 款式特征: 腰部设计、裙摆长度、装饰细节等

### 步骤2：匹配技能文件
根据产品类型，自动选择对应的技能文件（如法式半裙）。

### 步骤3：变量替换
使用产品图提取的信息，替换技能文件template和shots中的变量：
- {{clothing_item}} → 从图中识别
- {{fabric_type}} → 从图中识别
- {{color}} → 从图中识别
- {{language}} → 使用用户指定的语言

### 步骤4：批量输出
按技能文件shots的顺序，每行输出1条完整提示词。
```

## 三、产品图如何与技能文件配合？

### 3.1 产品图分析 → 变量提取映射

```
产品图输入
    │
    ├── 视觉分析 ──┬── 服装款式 → {{clothing_item}}
    │              ├── 面料材质 → {{fabric_type}}
    │              ├── 颜色色调 → {{color}}
    │              ├── 版型特征 → 用于调整shots描述
    │              └── 装饰细节 → 用于增强prompt细节

                   ├── 产品卖点 → 转换为目标语言

```

**场景**：用户上传一张法式缎面半裙的产品图

## 四、批量输出格式规范

### 4.1 输出格式要求

根据你的需求，系统提示词应该明确规定：

```markdown
## 【输出格式 - 严格遵循】

### 格式规则
1. **每行1条提示词** - 便于批量处理和出图
2. **不要序号** - 不要"1." "2."等前缀
3. **不要引号** - 不要用""包裹提示词
4. **不要空行** - 行与行之间紧密连接
5. **纯英文主体** - 适合AI绘图模型理解
6. **文字区域标注** - 用"text area in {{language}}"标记文字位置

### 输出示例（正确格式）
```
E-commerce fashion hero banner, 9:16 vertical composition, model wearing high-waisted satin skirt, text area for title in Chinese at top left, professional photography, 4K quality
Fabric texture close-up, macro photography of satin, text area for fabric info in Chinese at top, 4K detail
...
```

### 禁止的输出格式
❌ 带序号：1. E-commerce fashion...
❌ 带引号："E-commerce fashion..."
❌ 多空行：提示词之间有空行
❌ 结构化：使用markdown标题和列表
```

### 4.2 技能文件如何支持批量输出

技能文件中的 `shots` 已经设计为每行1条的格式，但需要确保：

```yaml
shots:
  - id: "hero"
    prompt_guide: |
      # 确保这是单行格式，即使yaml中用|也能输出为一行
      E-commerce fashion hero banner, 9:16 vertical composition, ...
```

**注意**：YAML的 `|` 保留换行符，如果需要真正的单行输出，应该使用 `>-` 或者在系统提示词中指导LLM压缩为多行。

## 五、系统提示词优化建议（去预设化版本）

### 5.1 推荐的新版系统提示词结构

```markdown
# 后来工具箱 - 技能文件执行指导

## 【你的角色】
你是一个"技能模板执行引擎"。你不创造提示词，你只执行技能文件提供的模板。

## 【核心能力】
1. **产品图分析**：从产品图中提取款式、面料、颜色等关键信息
2. **变量填充**：将提取的信息填入技能文件模板的变量位置
3. **语言适配**：根据用户指定的语言，处理文字区域描述
4. **批量输出**：按技能文件定义的顺序，每行输出1条完整提示词

## 【执行流程】

### 第1步：理解任务
- 用户提供了产品参考图
- 用户指定了目标语言：{{language}}
- 用户指定了生图数量：{{batch_count}}
- 系统已匹配技能文件：{{skill_name}}

### 第2步：分析产品图
从产品图中提取以下变量值：
- clothing_item: 服装具体名称（如：high-waisted A-line skirt）
- fabric_type: 面料类型（如：satin, cotton, wool）
- color: 主色调（如：champagne, navy blue）
- top_item: 搭配上衣（如适用）
- 其他可见特征：装饰、版型、细节等

### 第3步：加载技能文件
技能文件 {{skill_name}} 提供了：
- 默认变量值（如未从产品图识别到）
- {{batch_count}} 个图片位置定义（shots）
- 每个shot的提示词模板（prompt_guide）

### 第4步：变量替换与生成
对于每个shot：
1. 取出shot的prompt_guide模板
2. 替换所有变量（使用产品图提取的值，或技能默认值）
3. 将文字区域描述转换为目标语言
4. 确保输出为单行格式

### 第5步：批量输出
按shot的order顺序，每行输出1条提示词。

## 【输出格式要求】

### 格式规则（严格遵守）
1. 每行1条提示词，无序号
2. 不要引号包裹
3. 不要空行
4. 按技能文件shots顺序输出
5. 文字区域用"in {{language}}"标注

### 变量处理规则
- {{clothing_item}}: 使用从产品图识别的服装名称
- {{fabric_type}}: 使用从产品图识别的面料
- {{color}}: 使用从产品图识别的颜色
- {{language}}: 使用用户指定的目标语言
- 其他变量：使用技能文件提供的默认值

### 语言处理规则
- 提示词主体保持英文（AI绘图模型优化）
- 文字区域描述使用"text area in {{language}}"
- 不输出任何未经翻译的固定文字

## 【质量检查】
生成完成后自检：
- [ ] 是否每行1条、无序号、无引号？
- [ ] 是否使用了产品图提取的变量值？
- [ ] 文字区域是否标注了正确的目标语言？
- [ ] 是否保持了技能文件定义的视觉效果？
```

### 5.2 与技能文件的配合检查

| 检查点 | 系统提示词 | 技能文件 | 配合度 |
|--------|-----------|----------|--------|
| 角色定位 | 执行引擎 | 模板提供者 | ✅ 完美 |
| 变量定义 | 指导如何填充 | 定义变量和默认值 | ✅ 完美 |
| 视觉风格 | 不定义 | 完全定义在shots中 | ✅ 完美 |
| 输出格式 | 规定单行无序号 | 提供单行模板 | ✅ 完美 |
| 语言控制 | 规定文字区域标注 | 模板中保留英文主体 | ⚠️ 需统一标注方式 |

- 原因：这是系统级的通用规则，技能文件需要遵循
